# 14일 작업 일정

이 일정은 Unity 초보 3명이 2주 안에 플레이 가능한 MVP를 완성하기 위한 기준안입니다. 실제 일정은 팀원 가능 시간에 맞게 조정할 수 있지만, Day 12 이후 새 기능 추가 금지는 유지합니다.

## 운영 원칙

- 목표: 2주 안에 버그 없이 플레이 가능한 MVP
- 우선순위: 전체 플레이 루프 완성 > 안정성 > 그래픽 완성도 > 추가 기능
- 협업: 매일 짧은 점검 + 하루 1회 통합
- Scene 수정: 맵/Scene 담당 1명만 `MainScene` 수정
- 통합: 통합 담당이 진행하고 `main` merge 전 Unity Play Mode 확인
- Day 12 이후 새 기능 추가 금지

## 역할 약칭

- SYSTEM: 시스템 담당
- MAP: 맵/Scene 담당
- ASSET: 에셋/UI 담당
- LEAD: 통합/검수 담당

## Day 1-3: 기준 세팅과 이동

- SYSTEM: 필요한 스크립트와 Prefab 목록 작성, 이동 코드 시작
- MAP: `MainScene`, Grid/Tilemap, 도심 블록아웃 시작
- ASSET: 무료 에셋 후보와 직접 제작 스타일 기준 정리
- LEAD: repo/branch 세팅, Unity 실행 확인

완료 기준:
- Unity 프로젝트가 열리고 빈 `MainScene`이 실행된다.
- 플레이어가 도심 더미 맵에서 벽을 통과하지 않고 움직인다.

## Day 4-7: 도심 루프 완성

- SYSTEM: 수거, 가방, 분리수거, 오답 `-5`, 코인/정화율 계산
- MAP: 도심 20개 쓰레기 후보 위치, 수거함 1세트, 상점 더미 배치
- ASSET: 쓰레기 7종, 수거함 5종, HUD 아이콘 1차 제작
- LEAD: 도심 루프 Play Mode 확인과 중간 통합

완료 기준:
- 도심에서 수거 -> 분리수거 -> 코인/정화율 증가가 작동한다.
- 도심 1구역은 발표 가능한 수준으로 플레이된다.

## Day 8-11: 3구역 전체 루프 연결

- SYSTEM: Zone 데이터, Gate 개방, 특수 쓰레기, 업그레이드, 결과화면 연결
- MAP: 하천 20x15, 해안 20x15 간소 구역과 연결 통로 구성
- ASSET: 하천/해안 타일 보정, 특수 쓰레기, 특수 수거함, 결과화면 별 아이콘
- LEAD: 도심 -> 하천, 하천 -> 해안, 전체 루프 테스트

완료 기준:
- 정화율 100% 달성 시 다음 구역으로 진행할 수 있다.
- 시작부터 결과화면까지 전체 플레이가 한 번 이상 성공한다.

## Day 12-14: 안정화와 발표 준비

- 새 기능 추가 금지
- 치명 버그 수정
- 가방 가득 참, 오답, 코인 부족 같은 예외 상황 처리
- 플레이어가 끼는 위치 제거
- 시연 동선 최종 고정
- UI 가독성 수정
- 빌드 테스트와 백업 빌드 생성
- 발표 리허설과 실패 시 대체 시나리오 준비

완료 기준:
- 전체 루프가 2회 연속 성공한다.
- Unity Editor Play Mode가 성공한다.
- 빌드 실행이 성공한다.
- 발표용 백업본이 존재한다.

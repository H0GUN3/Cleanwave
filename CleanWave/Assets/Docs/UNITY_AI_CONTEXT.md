# Unity AI Context Mirror

이 문서는 Unity AI가 CleanWave 프로젝트 내부에서 바로 읽기 위한 미러 문서다. 최종 기준은 프로젝트 루트의 `PROJECT_CONTEXT.md`, `AGENTS.md`, `CleanWave_프로젝트_계획서.md`를 따른다.

## Project Summary

CleanWave는 오염된 `도심 -> 하천 -> 해안`을 순서대로 정화하는 힐링형 Unity 2D 환경 정화 게임이다. 플레이어는 연결형 1맵을 탐색하며 쓰레기를 줍고, 가방에 담고, 올바른 수거함에 배출해 구역 정화율을 올린다.

학습 방식은 상시 퀴즈가 아니라 직접 분리수거 중심이다. 다만 오분류가 발생했을 때만 퀴즈형 피드백 팝업으로 올바른 처리 방법을 안내한다.

## Core Loop

`탐색 -> 집게 수거 -> 가방 저장 -> 수거함 투입 -> 정답/오분류 판정 -> 피드백 -> 정화율 상승 -> 구역 개방 -> 코인/업그레이드`

1. 플레이어가 구역 안을 이동하며 쓰레기를 찾는다.
2. 집게로 쓰레기를 수거해 가방에 보관한다.
3. 수거함 근처에서 쓰레기를 일반 수거함 또는 특수 수거함에 배출한다.
4. 올바른 처리면 점수, 코인, 정화율이 오른다.
5. 오분류는 점수 `-5점`만 감점하며 별등급에는 영향을 주지 않는다.
6. 오분류 직후에는 문제/선택지/해설이 포함된 퀴즈형 피드백 팝업을 1회 표시한다.
7. 현재 구역 정화율이 `100%`가 되면 다음 구역이 열린다.
8. 획득 코인으로 플레이어 장비를 업그레이드한다.

## Fixed MVP Scope

- 맵 구조: 연결형 1맵, 3구역, 굽은 동선형
- 구역 순서: `도심 -> 하천 -> 해안`
- 다음 구역 해금: 현재 구역 정화율 `100%` 완료 후 개방
- 구역 크기: 구역당 `32x24 Tile`
- 쓰레기 수: 도심 `20개`, 하천 `25개`, 해안 `30개`
- 맵 제작 방식: Unity Tilemap + Sprite 오브젝트
- 수거함 배치: 구역마다 수거함 1세트
- MVP 포함: 직접 분리수거, 오분류 퀴즈형 피드백 팝업, 정화율, 업그레이드, 간단 결과화면

## Player Equipment Upgrades

업그레이드는 Clean Cart가 아니라 플레이어 장비 기준이다.

| 장비 | 역할 | 기본 | 중급 | 고급 | 비용 |
|---|---|---:|---:|---:|---|
| 집게 | 수거 거리 | 1칸 | 1.5칸 | 2칸 | 중급 50코인 / 고급 100코인 |
| 가방 | 보관 용량 | 5개 | 8개 | 12개 | 중급 50코인 / 고급 100코인 |
| 신발 | 이동 속도 | 100% | 115% | 130% | 중급 50코인 / 고급 100코인 |

## UI and Result Screen

HUD에는 아래 네 항목만 표시한다.

- 점수
- 코인
- 정화율
- 가방

결과화면에는 아래 세 항목만 표시한다.

- 점수
- 정화율
- 별등급

별등급 기준은 정화율 `70/80/95%`다.

오분류 퀴즈형 피드백 팝업은 상시 HUD 요소가 아니라 오분류 직후에만 표시되는 일회성 UI다. 팝업은 잘못 처리한 쓰레기에 대해 올바른 처리 방법을 묻고, 정답 해설로 왜 해당 수거함이 맞는지 설명한다.

팝업 구성은 `QuizFeedbackPanel`을 기준으로 하며, 반투명 배경, 둥근 픽셀 패널, 질문 아이콘, 문제 텍스트, 선택지 A/B, 해설 박스, 확인 버튼을 포함한다. 쓰레기와 수거함 아이콘은 새로 제작하거나 확정된 에셋을 작게 재사용하고, 문제/선택지/해설은 `Assets/Docs/QUIZ_BANK.md`의 텍스트를 TextMeshPro로 출력한다.

## Current Implementation Snapshot

- `Assets/Scenes/MainScene.unity`는 기본 URP 2D 씬에 가깝고, 주요 게임 오브젝트가 수동 배치된 완성 씬은 아니다.
- 확정 상수와 규칙은 `Assets/Scripts/Core/CleanWaveGameConstants.cs`와 `Assets/Scripts/Core/CleanWaveRules.cs`에 일부 반영되어 있다.
- 테스트용 런타임 생성 프로토타입과 기존 선택 에셋 묶음은 제거되었고, 새 타일/에셋 기반으로 MainScene을 구성할 예정이다.
- 플레이어, UI, 맵, 업그레이드 스크립트는 새 씬/프리팹 구조에 맞춰 본격 구현 전이다.
- `Assets/Prefabs`는 아직 본격적인 프리팹 기준선이 잡히기 전 상태다.

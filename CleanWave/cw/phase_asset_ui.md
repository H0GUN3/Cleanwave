# Draft: Asset and UI Phase Manual

## 담당 역할
- 플레이어, 쓰레기, 수거함, UI 리소스, 무료 맵 에셋 수집과 스타일 통일을 담당한다.
- 목표는 “2주 MVP에 필요한 최소 직접 제작 에셋과 UI 표시 리소스”를 안정적으로 제공하는 것이다.

## 범위 포함
- 플레이어 캐릭터 직접 제작.
- GPT 이미지 → volidless.dev → Aseprite → Unity 테스트 파이프라인 관리.
- 플레이어 Idle/Walk/Pickup 스프라이트 준비.
- 쓰레기 7종 16x16 제작.
- 수거함 5종 32x48 제작.
- HUD 아이콘/간단 UI 이미지 준비.
- 무료 맵 타일/배경 에셋 후보 수집.
- 파일명/폴더 규칙 준수.

## 범위 제외
- MainScene 직접 배치.
- 시스템 로직 코드 작성.
- 맵 구조 설계.
- 복잡한 UI 프로그래밍.
- 모든 맵 장식 직접 제작.

## Phase 0: 스타일 기준 확정
### 해야 할 일
- 스타듀식 32x32 탑다운 느낌을 기준으로 삼는다.
- 무료 에셋팩 후보를 1~2개로 제한한다.
- 직접 제작 에셋의 색감/테두리/명암 기준을 정한다.

### 산출물
- 참고 이미지 3~5개.
- 무료 에셋팩 후보 목록.
- 색감/스타일 기준 메모.

### 연결 지점
- 맵 담당: 사용할 타일팩 전달.
- 시스템 담당: Sprite 이름과 Prefab 연결 방식 확인.

### 확인 필요
- 무료 에셋 라이선스 사용 가능 여부.
- 모든 에셋을 같은 PPU/크기 기준으로 맞출 수 있는지.

### 주의사항
- 여러 스타일의 에셋팩을 섞지 않는다.
- 예쁜 에셋보다 Unity에 바로 넣기 쉬운 에셋을 우선한다.

## Phase 1: 플레이어 캐릭터 제작
### 해야 할 일
- 32x48 캐릭터 기준으로 GPT 이미지 생성.
- volidless.dev로 스프라이트화를 시도한다.
- Aseprite에서 프레임을 정리한다.
- 방향은 down/up/side 기준으로 만든다.
- side는 좌우 Flip으로 사용한다.

### 필요 프레임
- Idle down/up/side: 1~2프레임.
- Walk down/up/side: 각 4프레임.
- Pickup down/up/side: 각 3프레임.

### 완료 조건
- 캐릭터가 32x48 캔버스 안에 들어간다.
- 실제 몸 크기가 너무 크거나 작지 않다.
- Unity에서 걷기 방향이 구분 가능하다.
- Pickup 모션이 짧게라도 인식된다.

### 연결 지점
- 시스템 담당: Animator 파라미터 이름 확인.
- 맵 담당: 캐릭터가 32x32 타일 위에서 어색하지 않은지 확인.

### 확인 필요
- Animation Clip 이름.
- Pivot 위치.
- PPU 값.

### 주의사항
- 완벽한 애니메이션보다 구분 가능한 움직임을 우선한다.
- 생성 결과가 불안정하면 프레임 수를 줄인다.

### 대체안
- 캐릭터 제작 실패 시 무료 캐릭터 에셋 수정.
- Pickup 실패 시 집게 아이콘/반짝 이펙트로 대체.

## Phase 2: 쓰레기 7종 제작
### 해야 할 일
- 16x16 기준으로 쓰레기 7종 제작.
- 색 + 실루엣으로 구분한다.
- 종류: paper, can, plastic, vinyl, food, net, oil.

### 완료 조건
- 16x16에서도 종류가 어느 정도 구분된다.
- 파일명이 규칙에 맞다.
- Unity에서 너무 작아 보이지 않는다.

### 연결 지점
- 시스템 담당: TrashType 이름과 파일명이 대응되어야 한다.
- 맵 담당: 구역별 배치용 Sprite 전달.

### 확인 필요
- 각 쓰레기 타입 enum 이름.
- 쓰레기 Prefab을 개별로 만들지, Sprite만 교체할지.

### 주의사항
- 작은 크기에서는 디테일보다 실루엣이 중요하다.
- 색만 다른 복붙은 피한다.

## Phase 3: 수거함 5종 제작
### 해야 할 일
- 32x48 기준 수거함 5종 제작.
- 종류: paper, can, plastic, general, special.
- 색상과 간단한 아이콘/라벨로 구분한다.

### 완료 조건
- 5종이 한눈에 구분된다.
- 캐릭터 32x48과 나란히 있을 때 크기가 자연스럽다.
- 특수 수거함이 어망/기름 처리용으로 구분된다.

### 연결 지점
- 시스템 담당: BinType 이름과 Sprite 이름 대응.
- 맵 담당: 수거함 1세트 배치용 리소스 전달.

### 확인 필요
- 일반 수거함에 들어갈 쓰레기 타입.
- 특수 수거함 색상/아이콘.

### 주의사항
- 라벨 텍스트가 너무 작으면 아이콘 중심으로 표현한다.

## Phase 4: UI 리소스 준비
### 해야 할 일
- HUD 표시용 아이콘 준비: 점수, 코인, 정화율, 가방.
- 결과화면에 사용할 별 아이콘 준비.
- 오답 피드백용 간단 표시 리소스 준비.

### 완료 조건
- HUD 아이콘이 같은 스타일이다.
- 별등급 1/2/3 표시가 가능하다.
- UI 리소스가 시스템 담당에게 전달된다.

### 연결 지점
- 시스템 담당: HUD 텍스트/아이콘 연결.
- 문서/발표 담당: 결과화면 캡처에 사용할 UI 확인.

### 확인 필요
- UI를 이미지 중심으로 할지, TextMeshPro 텍스트 중심으로 할지.

### 범위 제외
- 복잡한 UI 애니메이션.
- 화려한 메뉴 화면.

## Phase 5: Unity 적용 점검
### 해야 할 일
- 제작한 에셋을 Unity에 넣어 크기/Pivot/PPU 확인.
- 이상하면 Aseprite에서 수정한다.
- 맵 담당과 Scene에서 크기 확인한다.

### 완료 조건
- 캐릭터, 쓰레기, 수거함이 같은 세계 안에 자연스럽게 보인다.
- Pivot 문제로 떠 보이거나 묻히지 않는다.
- 파일명이 규칙에 맞다.

### 연결 지점
- 맵 담당: 실제 배치 후 크기/가독성 피드백.
- 시스템 담당: Prefab 연결 후 Sprite 누락 확인.

### 주의사항
- 에셋 최종 수정은 Day 12 전까지 끝낸다.
- 이후에는 새 에셋 추가보다 누락/오류 수정만 한다.

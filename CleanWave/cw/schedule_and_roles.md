# Draft: Schedule and Roles

## 목적
- Unity 초보 3명이 2주 안에 플레이 가능한 MVP를 완성하기 위한 역할/일정 초안.

## 최종 목표
- 2주 후 플레이 가능한 MVP.
- 3구역 간소 구현.
- 우선순위: 버그 없이 플레이 > 그래픽 완성도 > 기능 수.

## 역할 분배 초안

### 1번: 시스템 담당
- 플레이어 이동.
- 쓰레기 수거.
- 가방 용량.
- 수거함 분리수거.
- 점수/코인/정화율.
- 구역 개방 로직.

### 2번: 맵/Scene 담당
- MainScene 수정 권한 보유.
- 32x32 Tilemap 구성.
- 도심/하천/해안 20x15 구역 배치.
- 수거함/상점/쓰레기 위치 배치.
- 팀원이 만든 맵 요소를 Scene에 통합.

### 3번: 에셋/UI 담당
- 플레이어 스프라이트 제작 파이프라인 관리.
- 쓰레기 7종 제작.
- 수거함 5종 제작.
- UI 아이콘/HUD 리소스 준비.
- 무료 맵 에셋 수집 및 스타일 통일.

### 사용자: 통합 담당
- Git merge 관리.
- main 실행 상태 확인.
- 매일 통합/점검 진행.
- 충돌/범위 증가 판단.

## 협업 방식
- 매일 짧은 점검.
- 하루 1회 통합.
- 통합 전 각자 브랜치에서 Unity 실행 확인.
- 통합 후 main에서 Unity Play Mode 확인.

## 14일 일정 초안
- Day 1: 프로젝트 생성, Git 세팅, 폴더 구조, 에셋 규칙 확정.
- Day 2: 더미 캐릭터/타일/쓰레기/수거함 Unity 테스트.
- Day 3: 플레이어 이동, 기본 맵 도심 구역 배치.
- Day 4: 쓰레기 수거, 가방 시스템 1차.
- Day 5: 수거함 분리수거, 오답 점수 감점.
- Day 6: 정화율/코인 계산, HUD 1차.
- Day 7: 도심 구역 플레이 루프 완성 및 중간 통합.
- Day 8: 하천/해안 간소 구역 복제·변형.
- Day 9: 구역 개방 로직, 특수 쓰레기/특수 수거함.
- Day 10: 업그레이드 1차, 결과화면 1차.
- Day 11: 3구역 전체 루프 연결.
- Day 12: 버그 수정, 그래픽 누락 보완, 발표용 플레이 경로 고정.
- Day 13: 새 기능 금지, QA/밸런스/빌드 확인.
- Day 14: 최종 발표 준비, 시연 리허설, 백업 빌드 생성.

## 미정
- 각 팀원 실제 작업 가능 시간.
- 일일 통합 시간.
- 발표 시연 기준 플레이 경로.

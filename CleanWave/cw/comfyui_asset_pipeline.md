# Draft: ComfyUI Asset Pipeline

## 목적
- CleanWave 플레이어 캐릭터 제작에 ComfyUI를 사용할지 판단하고, 사용할 경우 2주 MVP를 해치지 않는 범위를 정한다.

## 결론 초안
- ComfyUI는 최종 스프라이트 자동 생성 도구가 아니라 **컨셉/기준 프레임 생성 도구**로 사용하는 것이 안전하다.
- 최종 32x48 픽셀 스프라이트는 Aseprite 등에서 정리한다.
- 쓰레기 7종 x 3개, 수거함 5종, UI 아이콘은 직접 제작한다.

## 잘 맞는 용도
- 플레이어 캐릭터의 의상, 색감, 실루엣 탐색.
- 정면/측면/후면 기준 이미지 생성.
- 캐릭터 후보 여러 개 비교.
- 애니메이션 프레임을 만들기 전 참고 이미지 생성.

## 잘 안 맞는 용도
- 32x48 최종 픽셀 스프라이트를 한 번에 완성.
- Idle/Walk/Pickup 모든 프레임을 일관성 있게 자동 생성.
- 16x16 쓰레기 같은 초소형 오브젝트 제작.
- UI 아이콘 최종본 제작.

## 추천 파이프라인
1. 캐릭터 설명/스타일 기준 작성.
2. ComfyUI로 캐릭터 기준 이미지 생성.
3. down/up/side 방향 참고 이미지를 고른다.
4. Aseprite에서 32x48 캔버스로 픽셀 정리.
5. Idle/Walk/Pickup 프레임을 최소 프레임으로 제작.
6. Unity에서 크기, Pivot, 애니메이션 테스트.

## 2주 프로젝트용 사용 범위
- 사용: 플레이어 캐릭터 컨셉, 기준 포즈, 색감 후보.
- 제한적 사용: Pickup/Walk 프레임 참고 이미지.
- 사용하지 않음: 쓰레기 21개, 수거함 5개, HUD/UI 아이콘.

## 리스크
- 설치/모델 세팅에 시간이 들 수 있다.
- 프레임 간 얼굴/옷/비율이 흔들릴 수 있다.
- 진짜 픽셀아트 느낌은 수동 보정이 필요하다.
- 모델/LoRA 라이선스 확인이 필요하다.
- 결과물이 마음에 안 들면 일정이 밀릴 수 있다.

## 대체안
- Day 2까지 쓸 만한 기준 캐릭터가 안 나오면 무료 캐릭터 에셋 수정으로 전환한다.
- Pickup 애니메이션이 어려우면 집게 아이콘/반짝 이펙트로 대체한다.

## 결정 필요
- ComfyUI를 누가 세팅할지.
- 로컬 PC에서 돌릴지, 한 명이 생성 담당을 맡을지.
- 사용할 모델/LoRA 라이선스를 어떻게 기록할지.
- Day 2까지 실패하면 대체안으로 전환할지.

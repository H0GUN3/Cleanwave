# Draft: Map and Scene Phase Manual

## 담당 역할
- MainScene, Tilemap, 구역 배치, 수거함/상점/쓰레기 위치 배치, Scene 통합을 담당한다.
- 목표는 “3구역 간소 맵에서 시스템 담당 기능이 정상 작동할 수 있는 플레이 공간”을 만드는 것이다.

## 범위 포함
- `MainScene` 관리.
- 32x32 Tilemap 구성.
- 도심/하천/해안 구역당 20x15 Tile 배치.
- 구역 간 연결 통로 구성.
- 구역 잠금/개방 위치 구성.
- 수거함 1세트와 상점 위치 배치.
- 쓰레기 Prefab 배치.
- 충돌 타일/장애물 배치.
- 팀원들이 만든 Prefab/PNG/Tile을 Scene에 통합.

## 범위 제외
- 시스템 로직 코드 작성.
- 캐릭터 애니메이션 제작.
- 쓰레기/수거함 원본 스프라이트 제작.
- Git merge 최종 통합.
- 복잡한 맵 장식 완성도 작업.

## Phase 0: 맵 기준 준비
### 해야 할 일
- 32x32 Tile 기준으로 Grid/Tilemap을 설정한다.
- MainScene 이름과 위치를 고정한다.
- 맵 크기 기준을 확정한다: 각 구역 20x15 Tile.
- 도심 → 하천 → 해안 순서로 연결한다.

### 산출물
- 빈 MainScene.
- 3구역 블록아웃.
- 구역 경계/연결 통로 초안.

### 연결 지점
- 시스템 담당: 구역 개방 로직과 경계 오브젝트 이름을 맞춘다.
- 에셋 담당: 사용할 타일/장식 에셋팩을 전달받는다.

### 확인 필요
- Scene 파일 수정자는 본인 1명인지.
- Tilemap Layer 구조.
- Player 시작 위치.

### 주의사항
- 다른 팀원이 MainScene을 직접 수정하지 않도록 한다.
- 맵은 예쁘게 만들기보다 먼저 이동 가능한 구조를 만든다.

## Phase 1: 도심 구역 완성 기준 만들기
### 해야 할 일
- 도심 20x15 구역을 먼저 완성도 기준 구역으로 만든다.
- 시작 위치, 쓰레기, 수거함, 상점, 출구를 배치한다.
- 충돌 오브젝트를 배치한다.

### 완료 조건
- 플레이어가 도심 구역을 돌아다닐 수 있다.
- 쓰레기 20개를 배치할 수 있는 공간이 있다.
- 수거함 1세트가 접근 가능한 위치에 있다.
- 하천으로 가는 길이 잠금 상태로 존재한다.

### 연결 지점
- 시스템 담당: Trash Prefab, Bin Prefab, ZoneGate 구조 필요.
- 에셋 담당: 도심 타일/장식 에셋 필요.

### 확인 필요
- 도심 쓰레기 수: 20개.
- 수거함 간격.
- 상점 위치가 수거함 옆인지.

### 주의사항
- 맵이 좁아 보이면 장식보다 통로 확보를 우선한다.

## Phase 2: 하천/해안 간소 구역 복제·변형
### 해야 할 일
- 도심 구조를 참고해 하천 20x15, 해안 20x15 구역을 만든다.
- 하천은 물가/비닐 중심, 해안은 모래/어망/기름 중심으로 차별화한다.
- 장식 밀도는 낮게 유지한다.

### 완료 조건
- 하천에 쓰레기 25개 배치 가능.
- 해안에 쓰레기 30개 배치 가능.
- 구역별 특수 쓰레기 배치가 가능하다.
- 도심→하천→해안 순서로 이동 가능하다.

### 연결 지점
- 시스템 담당: 구역별 오염도 계산과 Trash 데이터 필요.
- 에셋 담당: 하천/해안 타일, 특수 쓰레기 Sprite 필요.

### 확인 필요
- 하천/해안의 정확한 연결 위치.
- 구역 개방 후 카메라 이동/플레이어 이동 방식.

### 주의사항
- 하천/해안은 도심보다 간소해도 된다.
- 전체 루프 연결을 장식 완성도보다 우선한다.

## Phase 3: Prefab 통합과 배치
### 해야 할 일
- 시스템 담당이 만든 Player/Trash/Bin/Gate Prefab을 Scene에 배치한다.
- 에셋 담당이 만든 Sprite를 Prefab에 반영한다.
- 쓰레기 위치를 구역별 수량에 맞춘다.

### 완료 조건
- 도심 20개, 하천 25개, 해안 30개 배치.
- 구역마다 수거함 1세트 배치.
- 상점 위치 배치.
- 플레이어 시작 위치 설정.

### 연결 지점
- 시스템 담당: Prefab 필드 누락 여부 확인 필요.
- 에셋 담당: Sprite 크기/Pivot 문제 확인 필요.

### 확인 필요
- Prefab을 Scene에 직접 복사할지, Spawner로 생성할지.
- 쓰레기별 타입 설정 방식.

### 주의사항
- 배치한 Prefab의 타입 값을 반드시 확인한다.
- Sprite만 바꾸고 Collider 크기를 안 맞추는 실수를 조심한다.

## Phase 4: 충돌/정렬/카메라 점검
### 해야 할 일
- Player가 벽/물/장애물을 통과하지 않는지 확인한다.
- 오브젝트 앞뒤 정렬이 이상하지 않은지 확인한다.
- 카메라가 전체 플레이에 적절한 크기인지 확인한다.

### 완료 조건
- 이동 불가능 영역을 통과하지 않는다.
- 쓰레기/수거함/플레이어가 시각적으로 크게 어긋나지 않는다.
- 플레이 경로가 막히지 않는다.

### 연결 지점
- 시스템 담당: Collider/Layer 충돌 설정 확인.
- 에셋 담당: Pivot/크기 수정 요청.

### 확인 필요
- Orthographic Size.
- Sorting Layer/Order 규칙.
- Y축 정렬 사용 여부.

### 범위 제외
- 완벽한 시각 연출.
- 복잡한 카메라 연출.

## Phase 5: 최종 플레이 루트 고정
### 해야 할 일
- 발표에서 보여줄 플레이 루트를 정한다.
- 쓰레기 배치를 너무 흩뿌리지 않고 시연 가능한 동선으로 조정한다.
- 버그 위험이 큰 장식/장애물을 제거한다.

### 완료 조건
- 시작 → 도심 정화 → 하천 개방 → 하천 정화 → 해안 개방 → 결과화면 흐름이 가능하다.
- 발표 시연자가 헤매지 않는다.

### 주의사항
- Day 12 이후에는 맵 구조를 크게 바꾸지 않는다.

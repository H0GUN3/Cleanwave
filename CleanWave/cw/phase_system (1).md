# Draft: System Phase Manual

## 담당 역할
- 플레이어 조작, 수거, 가방, 분리수거, 점수/코인, 정화율, 구역 개방 로직을 담당한다.
- 목표는 “3구역 간소 맵에서 처음부터 끝까지 플레이 가능한 루프”를 만드는 것이다.

## 범위 포함
- 플레이어 이동 입력 처리.
- 쓰레기 감지 및 수거 처리.
- 가방 용량 제한.
- 수거함 상호작용.
- 쓰레기 종류별 올바른 수거함 판정.
- 오분류 시 점수 -5점.
- 쓰레기 처리 시 코인 획득.
- 정화율 계산: 제거한 오염도 / 전체 오염도 × 100.
- 구역 정화율 100% 달성 시 다음 구역 개방.
- HUD에 점수/코인/정화율/가방 상태 전달.

## 범위 제외
- MainScene 직접 배치 수정.
- Tilemap 디자인 수정.
- 최종 캐릭터/쓰레기/수거함 스프라이트 제작.
- UI 디자인 이미지 제작.
- 발표 자료 작성.
- 미니맵, 랭킹, 콤보, 배터리, 튜토리얼, Clean Cart 구현.

## Phase 0: 준비
### 해야 할 일
- 프로젝트 폴더 구조 확인.
- `Scripts/` 하위에 시스템별 스크립트 위치를 정한다.
- 임시 더미 Sprite/Prefab으로 기능을 먼저 만든다.

### 산출물
- 시스템 스크립트 목록 초안.
- 필요한 Prefab 목록.

### 연결 지점
- 에셋/UI 담당에게 필요한 Sprite 목록 전달.
- 맵 담당에게 필요한 Prefab 이름과 배치 방식 전달.

### 확인 필요
- 플레이어 Prefab 이름.
- Trash Prefab 저장 위치.
- Bin Prefab 저장 위치.
- HUD 오브젝트 이름.

### 주의사항
- 실제 스프라이트가 없어도 기능을 먼저 더미로 완성한다.
- Scene 배치가 필요하면 맵 담당에게 요청한다.

## Phase 1: 플레이어 이동
### 해야 할 일
- 2D 탑다운 이동 구현.
- 상하좌우 4방향 입력 처리.
- 이동 방향 값을 Animator 또는 상태값으로 전달.
- 충돌체와 Rigidbody2D 설정 기준을 정한다.

### 완료 조건
- 플레이어가 32x32 Tilemap 위에서 끊김 없이 움직인다.
- 벽/장애물 충돌 시 통과하지 않는다.
- 이동 중 방향 상태가 `up/down/side`로 구분된다.

### 연결 지점
- 에셋 담당: `walk_up/down/side` 애니메이션 필요.
- 맵 담당: 충돌 타일/장애물 Layer 필요.

### 확인 필요
- 이동 속도 기본값.
- Tilemap Collider 사용 여부.
- Sorting 기준.

### 범위 제외
- 대시, 달리기, 스태미나.

## Phase 2: 쓰레기 수거 + 가방
### 해야 할 일
- 플레이어 주변 수거 가능 범위 감지.
- 상호작용 키로 쓰레기 수거.
- 쓰레기를 즉시 삭제하지 않고 가방 목록/카운트에 저장.
- 가방 용량 기본값 5개 적용.
- 가방이 가득 차면 추가 수거 불가 처리.

### 완료 조건
- 쓰레기 앞에서 상호작용 시 쓰레기가 맵에서 사라진다.
- 가방 카운트가 증가한다.
- 5개 초과 수거가 막힌다.
- 가방이 가득 찼을 때 간단한 피드백이 나온다.

### 연결 지점
- 에셋 담당: 쓰레기 7종 Sprite 필요.
- 맵 담당: Trash Prefab을 맵에 배치해야 함.
- UI 담당: 가방 카운트 표시 필요.

### 확인 필요
- 수거 키: 예: `F`.
- 수거 범위: 예: 플레이어 기준 1칸.
- 가방에 여러 종류를 섞어 담을지 여부.

### 주의사항
- 수거 판정은 애니메이션 프레임에 의존하지 않는다.
- Pickup 애니메이션이 없어도 기능은 작동해야 한다.

### 범위 제외
- 자동 수거.
- 수거 도구 업그레이드 자동 적용 외 추가 특수 능력.

## Phase 3: 수거함 분리수거
### 해야 할 일
- 수거함 5종 타입 정의: paper, can, plastic, general, special.
- 쓰레기 7종 타입 정의: paper, can, plastic, vinyl, food, net, oil.
- 쓰레기별 올바른 수거함 매핑 구현.
- 수거함 앞에서 상호작용 시 가방 안 쓰레기를 처리.
- 정답이면 점수/코인/정화율 반영.
- 오답이면 점수 -5점만 적용.

### 완료 조건
- 종이/캔/플라스틱은 각각 전용 수거함에 들어간다.
- 음식물/비닐은 일반 또는 정의된 매핑에 따라 처리된다.
- 어망/기름은 특수 수거함에서 처리된다.
- 오답 시 별등급/정화율에는 직접 영향 없이 점수만 감소한다.

### 연결 지점
- 에셋 담당: 수거함 5종 Sprite/Prefab 필요.
- 맵 담당: 각 구역에 수거함 1세트 배치 필요.
- UI 담당: 오답 피드백 표시 필요.

### 확인 필요
- 비닐/음식물의 최종 수거함 매핑.
- 가방 안 쓰레기를 하나씩 처리할지, 선택 UI를 둘지.
- UI 없이 가까운 수거함에 현재 선택 쓰레기만 처리할지.

### 주의사항
- 2주 MVP에서는 복잡한 인벤토리 UI를 피한다.
- 구현이 어려우면 “가방의 가장 오래된 쓰레기 1개 처리”로 단순화한다.

### 범위 제외
- 드래그 앤 드롭 인벤토리.
- 자동 분류 업그레이드.

## Phase 4: 정화율/코인/구역 개방
### 해야 할 일
- 각 구역의 전체 오염도 합계를 계산한다.
- 처리된 쓰레기의 오염도 합계를 누적한다.
- 정화율을 계산해 HUD에 전달한다.
- 구역 정화율 100% 시 다음 구역 문/길을 연다.
- 코인은 쓰레기 처리마다 지급한다.

### 완료 조건
- 도심 정화율 100% 후 하천 진입 가능.
- 하천 정화율 100% 후 해안 진입 가능.
- 전체 해안까지 완료 가능.
- 코인이 쓰레기별 보상 수치대로 증가한다.

### 연결 지점
- 맵 담당: 구역 경계/잠금 오브젝트 필요.
- UI 담당: 정화율 표시 필요.
- 문서 담당: 확정 수치와 코드 수치 일치 확인 필요.

### 확인 필요
- 구역 전환이 문 오브젝트인지, 콜라이더 해제인지.
- 구역별 쓰레기 배치 수가 계획과 일치하는지.

### 주의사항
- 정화율 계산은 쓰레기 개수가 아니라 오염도 기준이다.
- 하천/해안이 간소 구역이어도 루프는 끝까지 연결되어야 한다.

## Phase 5: 업그레이드/HUD/결과화면 연결
### 해야 할 일
- 장비 3종 업그레이드 수치 연결: 집게, 가방, 신발.
- 중급 50코인, 고급 100코인 비용 적용.
- HUD에 점수/코인/정화율/가방 표시.
- 결과화면에 점수/정화율/별등급 표시.

### 완료 조건
- 코인으로 업그레이드를 구매할 수 있다.
- 가방 업그레이드 후 용량이 5/8/12로 바뀐다.
- 신발 업그레이드 후 이동속도가 100/115/130%로 바뀐다.
- 집게 업그레이드 후 수거 범위가 1/1.5/2칸으로 바뀐다.
- 최종 결과화면이 표시된다.

### 연결 지점
- UI 담당: HUD/상점/결과화면 리소스 필요.
- 맵 담당: 상점 위치 배치 필요.

### 확인 필요
- 상점 UI를 별도 패널로 만들지, F키 즉시 업그레이드로 처리할지.
- 결과화면 표시 타이밍.

### 범위 제외
- 복잡한 상점 메뉴.
- 장비 외형 변화.

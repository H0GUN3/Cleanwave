## 3. 보완 방향

이 개정안은 환경 게임이라는 큰 방향만 두지 않고, 메타플로깅을 기준 개념으로 먼저 고정해 이후 상세 설계가 흔들리지 않게 한다.

차별화는 Spilled!과 겹치는 정화 소재가 아니라, 분리 수거 행동, 짧은 교육 피드백, 정화율 가중치로 만든다.

학습 구조는 상시 퀴즈 중심이 아니라 직접 분류하고 즉시 반응을 받는 흐름으로 잡아, 플레이와 학습이 동시에 이어지게 한다. 다만 오분류가 발생했을 때는 해당 쓰레기의 올바른 처리 방법을 알려주는 퀴즈형 피드백 팝업을 표시한다.

범위는 MVP에 필요한 정화, 업그레이드, 결과 확인만 남기고 배터리, 미니맵, 랭킹, 콤보, 튜토리얼, Clean Cart는 제외해 구현 밀도를 낮춘다.

구현 기준은 구역, 쓰레기, 장비, 수거함을 고정 표로 두어 개발자가 나중에 바로 작업 단위와 데이터를 읽을 수 있게 정리한다.

## 4. 핵심 플레이 루프

탐색 → 집게 수거 → 가방 저장 → 수거함 투입 → 정답/오분류 판정 → 피드백 → 정화율 상승 → 구역 개방 → 코인/업그레이드

1. 탐색: 구역을 이동하며 오염된 쓰레기를 찾는다.
2. 집게 수거: 집게로 쓰레기를 집어 가방에 담는다.
3. 가방 저장: 모은 쓰레기를 임시 보관한다.
4. 수거함 투입: 일반 수거함 또는 특수 수거함에 직접 넣는다.
5. 정답/오분류 판정: 수거함 타입과 쓰레기 타입을 비교한다.
6. 피드백: 정답이면 짧은 성공 피드백을 보여주고, 오분류면 퀴즈형 팝업으로 올바른 처리 방법을 안내한다.
7. 정화율 상승: 올바른 처리량이 누적되며 구역 정화율이 오른다.
8. 구역 개방: 정화율 100%가 되면 다음 구역이 열린다.
9. 코인/업그레이드: 코인을 모아 집게, 가방, 신발을 강화한다.

## 5. 교육 방식

이 게임의 학습은 상시 퀴즈 중심이 아니라, 쓰레기를 직접 분류하는 행동 자체로 이루어진다.

플레이어는 쓰레기를 해당 일반 수거함이나 특수 수거함에 바로 넣고, 그 결과를 피드백으로 확인한다. 정답 처리에는 짧은 성공 피드백을 표시하고, 오분류에는 문제/선택지/해설이 포함된 퀴즈형 피드백 팝업을 표시한다.

오분류 퀴즈형 피드백 팝업은 오분류 직후에만 발생한다. 플레이어가 잘못 버린 쓰레기에 대해 올바른 처리 방법을 묻고, 해설로 왜 그 수거함이 맞는지 알려준다. 오분류가 발생해도 결과는 점수 `-5점`만 적용하고, 별등급에는 영향을 주지 않는다.

전체 게임오버는 없고, 구역 재시도도 없으며, 진행은 끝까지 힐링형 정화 흐름을 유지한다.

## 6. 연결형 맵 설계

MVP의 맵 구조는 독립 Stage1, Stage2, Stage3 씬이 아니라 `연결형 1맵`으로 구성한다. 전체 진행은 도심 → 하천 → 해안으로 이어지는 굽은 동선형 흐름이며, 구역 간 이동은 한 맵 안에서 자연스럽게 이어진다.

맵 제작은 `Unity Tilemap + Sprite 오브젝트` 조합을 기준으로 한다. Tilemap은 바닥, 지형, 동선을 정리하고, Sprite 오브젝트는 쓰레기, 수거함, 상점, 장비 배치처럼 개별 상호작용 요소를 배치하는 방식으로 사용한다.

각 구역은 `32x24 Tile` 규모로 설계한다. 기존 `20x15 Tile`은 초기 블록아웃 검토 기준이었으나, 플레이어 캐릭터 `32x48` 크기와 쓰레기/수거함/상점/게이트 배치 밀도를 고려해 구역별 실제 플레이 공간은 `32x24 Tile`로 확장한다. 쓰레기 수는 도심 `20개`, 하천 `25개`, 해안 `30개`로 고정하며, 각 구역마다 `수거함 1세트`를 둔다. 또한 각 구역의 수거함 주변에는 집게, 가방, 신발 업그레이드를 위한 장비 상점을 배치한다.

| 구역 | 크기 | 쓰레기 수 | 주요 특징 | 배치 요소 |
|---|---|---|---|---|
| 도심 | 32x24 Tile | 20개 | 출발 구역, 생활 쓰레기 중심, 넓은 중심 동선 | 수거함 1세트, 장비 상점, 집게/가방/신발 업그레이드, 하천 잠금 게이트 |
| 하천 | 32x24 Tile | 25개 | 도심에서 이어지는 중간 구역, 물가 흐름과 곡선 산책로 반영 | 수거함 1세트, 장비 상점, 집게/가방/신발 업그레이드, 해안 잠금 게이트 |
| 해안 | 32x24 Tile | 30개 | 최종 구역, 해안 오염 집중, 가장 열린 공간감 | 수거함 1세트, 장비 상점, 집게/가방/신발 업그레이드 |

이 설계에서 각 구역은 개별 씬처럼 분리되지 않고, 하나의 맵 안에서 정화율 100% 달성 시 다음 구역이 열리는 방식으로 연결된다.

### 맵 구조 도식

```text
┌────────────────────────────┐
│ zone_city / 도심 32x24      │
│ 시작 위치                   │
│ 도로·인도·건물·생활 쓰레기  │
│ 쓰레기 20개                 │
│ 수거함 1세트 + 장비 상점    │
└───────────────┬────────────┘
                │ 도심 100% 정화 후 개방
                ▼
        [하천 잠금 게이트]
                │
┌───────────────┴────────────┐
│ zone_river / 하천 32x24     │
│ 강변 산책로·물가·나무·바위  │
│ 비닐 중심 특수 쓰레기       │
│ 쓰레기 25개                 │
│ 수거함 1세트 + 장비 상점    │
└───────────────┬────────────┘
                │ 하천 100% 정화 후 개방
                ▼
        [해안 잠금 게이트]
                │
┌───────────────┴────────────┐
│ zone_coast / 해안 32x24     │
│ 모래사장·바닷물·부두·바위   │
│ 어망/기름 중심 특수 쓰레기  │
│ 쓰레기 30개                 │
│ 수거함 1세트 + 장비 상점    │
└────────────────────────────┘
```

전체 형태는 직선 복도가 아니라 `S자형` 또는 완만한 `ㄱ자 + 곡선` 흐름으로 구성한다. 각 구역 안에는 메인 동선 1개, 짧은 우회 동선 1~2개, 쓰레기 수거 포켓, 수거함/상점 랜드마크를 둔다. 장식 밀도보다 플레이어 이동 여백과 수거 동선의 가독성을 우선한다.

## 7. 정화율·쓰레기·보상 기준

정화율은 단순히 처리한 쓰레기 개수가 아니라, 각 쓰레기의 오염도를 가중치로 반영해 계산한다. 따라서 같은 개수라도 오염도가 높은 쓰레기를 더 많이 제거하면 정화율이 더 크게 오른다.

정화율 = 제거한 오염도 / 전체 오염도 × 100

| 쓰레기 | 오염도 | 획득 코인 | 비고 |
|---|---:|---:|---|
| 종이 | 1 | 1 | 기본 생활 쓰레기 |
| 캔 | 2 | 2 | 기본 생활 쓰레기 |
| 플라스틱 | 3 | 3 | 기본 생활 쓰레기 |
| 비닐 | 3 | 3 | 하천 특수 쓰레기 |
| 음식물 | 3 | 3 | 도심 특수 쓰레기 |
| 어망 | 5 | 5 | 해안 특수 쓰레기 |
| 기름 | 5 | 5 | 해안 특수 쓰레기 |

구역별 쓰레기 배치는 완전한 랜덤 스폰이 아니라, 아래의 구역별 특수 쓰레기 기준을 우선으로 삼아 배치한다. 이 기준을 바탕으로 각 구역의 분위기와 학습 메시지를 맞춘다.

| 구역 | 특수 쓰레기 |
|---|---|
| 도심 | 음식물 |
| 하천 | 비닐 |
| 해안 | 어망 / 기름 |

## 8. 장비·업그레이드 명세

장비는 Clean Cart 기준이 아니라 플레이어 기준으로 적용한다. 즉, 플레이어가 직접 집게, 가방, 신발을 강화하며 이동과 수거 편의성을 높인다.

| 장비 | 역할 | 기본 | 중급 | 고급 | 비용 |
|---|---|---|---|---|---|
| 집게 | 수거 거리 | 1칸 | 1.5칸 | 2칸 | 중급 50코인 / 고급 100코인 |
| 가방 | 보관 용량 | 5개 | 8개 | 12개 | 중급 50코인 / 고급 100코인 |
| 신발 | 이동 속도 | 100% | 115% | 130% | 중급 50코인 / 고급 100코인 |

### 업그레이드 상점

각 구역에는 분리수거장 옆에 장비 상점이 배치된다. 상점은 집게, 가방, 신발을 각각 담당하는 오브젝트로 나뉘며, 플레이어가 관련 상점 근처에 서서 `F키`를 누르면 코인이 즉시 차감되고 해당 장비가 바로 업그레이드된다.

## 9. HUD 및 결과화면

HUD에는 `점수 / 코인 / 정화율 / 가방`만 표시한다. 배터리, 미니맵, 랭킹, 콤보, 튜토리얼 같은 제외 시스템은 HUD에 표시하지 않는다.

오분류 퀴즈형 피드백 팝업은 HUD 상시 요소가 아니라, 쓰레기를 잘못된 수거함에 넣은 직후에만 표시되는 일회성 피드백 UI다. 팝업에는 문제, 선택지, 정답 해설을 포함해 올바른 처리 방법을 안내한다.

결과화면에는 `점수 + 정화율 + 별등급`만 표시한다. 플레이 결과는 이 세 항목으로만 요약해 보여준다.

## 10. 문서 양식 및 개발툴 정의

### 문서 양식

이 문서는 `개발 명세형` 양식으로 작성한다. 이 프로젝트는 구현 전에 확정해야 할 항목이 많고, 개발자가 바로 읽고 작업할 수 있는 결정값, 표, WBS가 필요하므로 개요 문서보다 명세 문서에 가깝다.

### 핵심 개발툴

핵심 개발툴은 MVP 계획에 직접 필요한 도구만 적는다. `Blender`, `Photoshop`, `Jira`, `Trello`, 서버, 온라인 랭킹 도구는 이 MVP 계획의 핵심 도구가 아니다.

| 도구 | 용도 | 사용 범위 |
|---|---|---|
| Unity 6.3 LTS | Unity 2D 프로젝트 제작, Tilemap 기반 맵 구성, Sprite 오브젝트 배치 | 본 프로젝트의 메인 제작 환경 |
| Visual Studio 또는 Rider | C# 스크립트 작성과 디버깅에 사용할 IDE 후보 | 코드 작성과 문제 확인용 |
| GitHub | 버전 관리와 팀 작업 기록 관리 | 소스와 문서 이력 관리용 |
| 무료 에셋 | 배경, 오브젝트, 사운드의 초기 제작 부담 완화 | MVP 초반 리소스 보강용 |
| Aseprite | 2D 스프라이트 수정과 간단한 픽셀/오브젝트 에셋 제작 | 소규모 2D 에셋 편집용 |
| Figma | UI/HUD 배치, 간단한 화면 흐름 정리 | 화면 구조와 UI 정리용 |

## 11. 작업분해표(WBS)

이 WBS는 실제 Unity 구현이 아니라 문서와 설계 명세를 정리하기 위한 작업분해표다.

| 구분 | 작업 단위 | 주요 내용 | 산출물 | 검증 기준 |
|---|---|---|---|---|

### MVP / 제외 / 향후 확장 구분

MVP는 `연결형 1맵`, `3구역`, `직접 분리수거`, `오분류 퀴즈형 피드백 팝업`, `정화율`, `업그레이드`, `결과화면`만 포함한다.

제외는 `배터리`, `미니맵`, `랭킹`, `콤보`, `튜토리얼`, `Clean Cart`로 고정한다.

향후 확장은 `추가 구역`, `고급 연출`, `추가 교육 콘텐츠`처럼 이후에 넓힐 수 있는 범위로만 남긴다.

## 12. 최종 확정 사항 및 미결정 사항

현재까지 논의된 핵심 항목은 모두 확정되었으며, 이 문서는 추가적인 추상 선택지를 조용히 채워 넣지 않는다. 아래 항목은 최종 기준으로 고정한다.

- 오답 감점값: `-5점`
- 오분류 피드백: 오분류 직후 `퀴즈형 팝업`으로 올바른 처리 방법 안내
- 어망/기름 처리: `특수 수거함`
- 구역 재시도: `없음`
- 다음 구역 개방: `정화율 100% 완료 후 개방`

### 미결정 사항

현재 문서 범위에서 논의된 핵심 항목 중 남아 있는 미결정 사항은 없다. 다만 `추가 구역`, `고급 연출`, `추가 교육 콘텐츠` 같은 향후 확장 항목을 실제로 추진할 경우, 해당 확장 범위의 세부 결정은 별도 문서나 후속 회의에서 따로 확정해야 한다.

## 13. 검증 체크리스트

- [ ] 확정 상수 확인: 구역 순서는 `도심 → 하천 → 해안`이며, 다음 구역은 `100% 완료 후 개방`으로 유지된다.
- [ ] 확정 상수 확인: 별등급 기준 `70/80/95%`, 집게 `1/1.5/2`, 가방 `5/8/12`, 신발 `100/115/130%`, 중급 `50코인`, 고급 `100코인`이 문서 전체에서 일관된다.
- [ ] 제외 시스템 확인: `배터리`, `미니맵`, `랭킹`, `콤보`, `튜토리얼`, `Clean Cart`는 MVP에 포함하지 않는다.
- [ ] 범위 충실도 확인: 이 문서는 프로젝트 계획과 설계 명세를 정리하는 문서이며, Unity 구현 작업 지시를 포함하지 않는다.
- [ ] 교육 충실도 확인: 학습은 상시 퀴즈 중심이 아니라 직접 분리수거 중심으로 유지되며, 퀴즈형 팝업은 오분류 직후 피드백으로만 사용된다.
- [ ] 맵 충실도 확인: 맵은 독립 Stage 씬 묶음이 아니라 연결형 1맵 구조로 유지된다.

이 문서는 프로젝트 계획과 작업 배분의 기준으로 사용할 준비가 완료되었다.
